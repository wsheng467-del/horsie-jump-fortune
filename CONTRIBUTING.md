# Contributing to Horsie Jump Fortune

Thank you for your interest in contributing! We welcome code, art, design, localization, and documentation contributions.

How to contribute

1. Issues
- Please open an issue to discuss significant changes before working on them.
- Use clear titles and a description of the problem or requested feature.

2. Pull Requests
- Fork the repository and create a branch for your changes: `git checkout -b feature/your-feature`.
- Keep changes focused and document the intent in the PR description.
- Follow the code style used in the repository.
- Write or update tests where applicable.

3. Code Style
- Keep code readable and well-documented.
- Follow existing naming conventions and project structure.

4. Assets (art, audio)
- Provide sources and attribution for any third-party assets.
- Preferably include optimized formats for web and mobile (e.g., PNG, WebP, OGG, etc.).

5. Localization
- For translations, provide text files or separate language READMEs.

6. CI and Tests
- Ensure CI passes for your PRs. We use GitHub Actions to run builds and tests.

License
- By contributing, you agree that your contributions will be licensed under the project's LICENSE file (e.g., MIT).

Thank you for helping improve Horsie Jump Fortune!
